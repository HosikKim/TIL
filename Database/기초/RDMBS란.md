# RDBMS (Relational Database Management System)

## 개념

**관계형 데이터베이스 관리 시스템**: 테이블(관계) 형태로 데이터를 저장하고 관리하는 시스템

### 핵심 특징
- **테이블 구조**: 행(Row)과 열(Column)로 데이터 구성
- **관계**: 테이블 간 외래키로 연결
- **SQL 사용**: 표준화된 질의 언어
- **ACID 보장**: 트랜잭션의 안정성 확보

## 장점

### 1. 데이터 무결성
- 기본키(PK), 외래키(FK), 제약조건으로 정확성 보장
- 중복 데이터 최소화

### 2. 강력한 쿼리 기능
- SQL로 복잡한 데이터 검색 및 조작 가능
- 조인, 집계, 서브쿼리 등 다양한 연산 지원

### 3. 트랜잭션 관리
- ACID 속성으로 데이터 일관성 유지
- 동시성 제어

### 4. 보안
- 사용자별 권한 관리
- 데이터 접근 제어

### 5. 데이터 독립성
- 물리적/논리적 구조 분리
- 스키마 변경 시 응용 프로그램 영향 최소화

## 단점

### 1. 확장성 제한
- 수평 확장(Scale-out) 어려움
- 대용량 데이터 처리 시 성능 저하

### 2. 경직된 스키마
- 사전에 정의된 스키마 필요
- 구조 변경 시 비용 큼

### 3. 성능 이슈
- 복잡한 조인 연산 시 속도 저하
- 정규화로 인한 다중 테이블 접근

### 4. 비정형 데이터 부적합
- JSON, 이미지, 로그 등 처리 제한적
- 고정된 스키마로 유연성 부족

## RDBMS vs NoSQL

| 구분 | RDBMS | NoSQL |
|------|-------|-------|
| 데이터 모델 | 테이블(관계형) | 문서/키-값/그래프 등 |
| 스키마 | 고정 | 유연 |
| 확장성 | 수직 확장 | 수평 확장 |
| 트랜잭션 | ACID 보장 | BASE (일부 제한) |
| 적합한 경우 | 정형 데이터, 복잡한 쿼리 | 비정형 데이터, 대용량 |

## 선택 기준

**RDBMS 사용 시**:
- 데이터 일관성이 중요한 경우 (금융, 회계)
- 복잡한 관계 표현 필요
- ACID 트랜잭션 필수

**NoSQL 고려 시**:
- 대규모 분산 시스템
- 스키마 변경 빈번
- 실시간 빅데이터 처리
