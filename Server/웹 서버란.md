# 웹 서버 (Web Server)

## 개념

클라이언트의 HTTP 요청을 받아 **정적 콘텐츠**를 제공하는 서버

### 역할
- HTML, CSS, JS, 이미지 등 정적 파일 제공
- HTTP 요청/응답 처리
- 클라이언트와 서버 간 통신

## 웹 서버 vs WAS

| 구분 | 웹 서버 (Web Server) | WAS (Web Application Server) |
|------|---------------------|------------------------------|
| 역할 | 정적 콘텐츠 제공 | 동적 콘텐츠 생성 + 정적 제공 |
| 처리 | HTML, CSS, JS, 이미지 | 비즈니스 로직, DB 연동 |
| 예시 | Apache, Nginx | Tomcat, JBoss, WebLogic |

### 동작 방식

**웹 서버만 사용**:
```
클라이언트 → 웹 서버 → 정적 파일 반환
```

**웹 서버 + WAS**:
```
클라이언트 → 웹 서버 → WAS (동적 처리) → 웹 서버 → 클라이언트
```

## 주요 웹 서버

### Apache HTTP Server
- 오픈소스
- 가장 널리 사용
- 모듈 확장 가능
- `.htaccess` 지원

### Nginx
- 고성능, 경량
- 비동기 이벤트 기반
- 리버스 프록시, 로드 밸런싱
- 정적 콘텐츠 처리 빠름

### Tomcat (WAS)
- Java Servlet/JSP 실행
- 웹 서버 기능 포함
- Spring Boot 내장 서버

## 웹 서버 vs WAS 비교

| 항목 | Apache/Nginx | Tomcat |
|------|--------------|--------|
| 분류 | 웹 서버 | WAS |
| 정적 처리 | 매우 빠름 ⭐⭐⭐ | 보통 ⭐⭐ |
| 동적 처리 | 불가 (위임 필요) | 가능 ⭐⭐⭐ |
| 용도 | 정적 파일, 프록시 | Java 애플리케이션 |

## 구성

### 일반적인 구조
```
클라이언트
  ↓
Nginx (웹 서버) - 정적 파일 처리, 로드 밸런싱
  ↓
Tomcat (WAS) - 동적 처리, 비즈니스 로직
  ↓
Database
```

### Spring Boot (개발 환경)
```
클라이언트
  ↓
내장 Tomcat - 웹 서버 + WAS 통합
  ↓
Database
```

## 선택 가이드

**웹 서버 (Apache/Nginx) 사용**:
- 정적 파일만 제공
- 리버스 프록시 필요
- 로드 밸런싱

**WAS (Tomcat) 사용**:
- Java 애플리케이션
- 동적 콘텐츠 생성
- Spring Boot

**웹 서버 + WAS**:
- 대규모 트래픽
- 정적/동적 분리
- 보안 강화
