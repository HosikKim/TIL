# MCP란?

## MCP (Model Context Protocol) 개요

MCP는 Anthropic에서 개발한 오픈 프로토콜로, AI 애플리케이션이 외부 데이터 소스와 도구에 안전하게 접근할 수 있도록 하는 표준화된 방법입니다.

### 왜 MCP가 필요한가?

기존에는 AI 모델마다 각각 다른 방식으로 외부 시스템과 통합해야 했습니다. MCP는 이를 표준화하여:
- 한 번 구축한 통합을 여러 AI 애플리케이션에서 재사용 가능
- 데이터 소스와 도구를 쉽게 추가하거나 제거
- 보안과 권한 관리를 일관되게 처리

## MCP의 핵심 구성 요소

### 1. **Resources (리소스)**
AI 모델이 접근할 수 있는 데이터나 콘텐츠
- 파일, 데이터베이스 레코드, API 응답 등
- 예: 프로젝트 파일, 문서, 데이터베이스 쿼리 결과

### 2. **Prompts (프롬프트)**
사전 정의된 프롬프트 템플릿
- 자주 사용하는 작업을 템플릿화
- 예: "이 코드를 리뷰해줘", "이 데이터를 분석해줘"

### 3. **Tools (도구)**
AI 모델이 실행할 수 있는 함수나 작업
- 외부 시스템과의 상호작용
- 예: 파일 생성, 데이터베이스 쿼리, API 호출

## MCP 작동 방식

```
┌─────────────┐         ┌─────────────┐         ┌──────────────┐
│             │         │             │         │              │
│  AI Client  │◄───────►│ MCP Server  │◄───────►│  Data Source │
│  (Claude)   │  MCP    │             │         │  (File, DB)  │
│             │Protocol │             │         │              │
└─────────────┘         └─────────────┘         └──────────────┘
```

### 동작 과정

1. **연결 설정**: AI 클라이언트(예: Claude Desktop)가 MCP 서버에 연결
2. **기능 탐색**: 서버가 제공하는 리소스, 프롬프트, 도구 목록 확인
3. **요청/응답**: 클라이언트가 필요한 기능 호출, 서버가 처리 후 응답
4. **실행**: AI가 받은 데이터를 활용하여 작업 수행

### 통신 방식

- **프로토콜**: JSON-RPC 2.0 기반
- **전송 방식**: 
  - stdio (표준 입출력) - 로컬 프로세스
  - HTTP/SSE - 원격 서버
- **메시지 형식**: JSON

## MCP의 장점

### 개발자 관점
- 표준화된 인터페이스로 개발 시간 단축
- 한 번 작성한 서버를 여러 AI 클라이언트에서 재사용
- 명확한 보안 경계 설정

### 사용자 관점
- AI가 필요한 컨텍스트에 안전하게 접근
- 다양한 도구와 데이터를 통합하여 활용
- 개인 데이터 제어 가능

## 실제 사용 예시

```javascript
// MCP 서버가 제공하는 기능 예시
- GitHub 저장소 읽기/쓰기
- 로컬 파일 시스템 접근
- 데이터베이스 쿼리
- 슬랙 메시지 전송
- Google Drive 파일 검색
```

## 주의 해야할점(보안)
MCP는 REST처럼 JSON/HTTP 기반의 프로토콜을 정의한 스펙이다. 즉 통신 규약만을 정의하지 실제로 애플리케이션을 구현하기 위한 많은 부분이 생략 되어있다. 특히 보안 인증, 인가나 에러 메시지와 같은 부분이 생략되어 있기 때문에 단순하게 MCP server를 구현하게 되면, 보안이나 기타 운영 부분에 많은 부분이 부족할 수 있다.
