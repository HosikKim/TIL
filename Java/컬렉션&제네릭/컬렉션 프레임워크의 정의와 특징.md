# 컬렉션 프레임워크

## 개념

데이터 그룹을 저장하고 조작하기 위한 표준화된 구조

**위치**: java.util 패키지

## 주요 인터페이스

### 1. List (순서 O, 중복 O)

**특징**:
- 인덱스로 접근
- 순서 유지
- 중복 허용

**구현 클래스**:

| 클래스 | 특징 | 용도 |
|--------|------|------|
| ArrayList | 배열 기반, 빠른 조회 | 조회 많을 때 |
| LinkedList | 연결 리스트, 빠른 삽입/삭제 | 삽입/삭제 많을 때 |
| Vector | 동기화 지원 | 멀티스레드 |
```java
List<String> list = new ArrayList<>();
list.add("A");
list.add("B");
list.add("A");  // 중복 가능
list.get(0);    // 인덱스 접근
```

### 2. Set (순서 X, 중복 X)

**특징**:
- 중복 불허
- 순서 없음 (LinkedHashSet 제외)

**구현 클래스**:

| 클래스 | 특징 | 용도 |
|--------|------|------|
| HashSet | 빠른 성능 | 일반적 사용 |
| TreeSet | 정렬된 순서 | 정렬 필요 |
| LinkedHashSet | 입력 순서 유지 | 순서 필요 |
```java
Set<String> set = new HashSet<>();
set.add("A");
set.add("B");
set.add("A");  // 중복 무시
// set.get(0);  // 인덱스 접근 불가
```

### 3. Map (키-값 쌍)

**특징**:
- 키 중복 불가
- 값 중복 가능

**구현 클래스**:

| 클래스 | 특징 | 용도 |
|--------|------|------|
| HashMap | 빠른 성능 | 일반적 사용 |
| TreeMap | 키 정렬 | 정렬 필요 |
| LinkedHashMap | 입력 순서 유지 | 순서 필요 |
```java
Map<String, Integer> map = new HashMap<>();
map.put("A", 1);
map.put("B", 2);
map.get("A");  // 1
```

### 4. Queue (FIFO)

**특징**:
- 선입선출
- 순서 중요

**구현 클래스**:

| 클래스 | 특징 |
|--------|------|
| LinkedList | 일반 큐 |
| PriorityQueue | 우선순위 큐 |
```java
Queue<String> queue = new LinkedList<>();
queue.offer("A");  // 추가
queue.poll();      // 제거 및 반환
```

## 주요 메서드

| 메서드 | List | Set | Map | Queue |
|--------|------|-----|-----|-------|
| 추가 | add() | add() | put() | offer() |
| 제거 | remove() | remove() | remove() | poll() |
| 조회 | get() | - | get() | peek() |
| 크기 | size() | size() | size() | size() |
| 비었는지 | isEmpty() | isEmpty() | isEmpty() | isEmpty() |
| 전체삭제 | clear() | clear() | clear() | clear() |

## 선택 기준

| 요구사항 | 선택 |
|---------|------|
| 순서 + 중복 허용 | List |
| 중복 불허 | Set |
| 키-값 쌍 | Map |
| FIFO 처리 | Queue |
| 빠른 조회 | ArrayList, HashMap |
| 빠른 삽입/삭제 | LinkedList |
| 정렬 필요 | TreeSet, TreeMap |

## 성능 비교

### List 성능

| 작업 | ArrayList | LinkedList |
|------|-----------|-----------|
| 조회 | O(1) 빠름 | O(n) 느림 |
| 삽입/삭제 (끝) | O(1) | O(1) |
| 삽입/삭제 (중간) | O(n) | O(1) |

### Set/Map 성능

| 작업 | HashSet/HashMap | TreeSet/TreeMap |
|------|-----------------|-----------------|
| 추가 | O(1) | O(log n) |
| 조회 | O(1) | O(log n) |
| 정렬 | X | O (자동) |

## 제네릭 활용
```java
// 타입 안정성
List<String> list = new ArrayList<>();
list.add("문자열");
// list.add(123);  // 컴파일 에러

// 타입 변환 불필요
String str = list.get(0);  // 캐스팅 불필요
```

## 정리

**컬렉션 사용 이유**:
- 효율적 데이터 관리
- 표준화된 API
- 제네릭으로 타입 안전

**선택 가이드**:
- 순서 중요 → List
- 중복 제거 → Set
- 키-값 → Map
- 처리 순서 → Queue
