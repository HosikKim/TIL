# Java 데이터 타입

## 개념

변수가 저장할 수 있는 값의 종류와 범위를 정의

## Primitive Type (원시 타입)

**정의**: 값 자체를 직접 저장하는 기본 데이터 타입

### 종류

| 분류 | 타입 | 크기 | 범위 | 기본값 |
|------|------|------|------|--------|
| 정수형 | byte | 1byte | -128 ~ 127 | 0 |
| | short | 2byte | -32,768 ~ 32,767 | 0 |
| | int | 4byte | -21억 ~ 21억 | 0 |
| | long | 8byte | 약 -9경 ~ 9경 | 0L |
| 실수형 | float | 4byte | 소수점 6-7자리 | 0.0f |
| | double | 8byte | 소수점 15자리 | 0.0d |
| 문자형 | char | 2byte | 유니코드 문자 | '\u0000' |
| 논리형 | boolean | 1bit | true / false | false |

### 특징
- Stack 메모리에 값 직접 저장
- null 할당 불가
- 빠른 처리 속도
- 메모리 효율적

## Reference Type (참조 타입)

**정의**: 객체의 주소를 저장하는 타입

### 종류
- **클래스**: 사용자 정의 객체
- **인터페이스**: 메서드 집합 정의
- **배열**: 동일 타입 요소 집합
- **열거형**: 상수 집합
- **기타**: String, List, Map, Set

### 특징
- Heap에 객체 저장, Stack에 주소 저장
- null 할당 가능
- 객체 재사용 가능
- 기본값: null

## Primitive vs Reference

| 구분 | Primitive | Reference |
|------|-----------|-----------|
| 저장 위치 | Stack | Stack(주소) + Heap(객체) |
| 저장 내용 | 값 | 주소 |
| 크기 | 고정 | 가변 |
| null | 불가 | 가능 |
| 속도 | 빠름 | 느림 |
| 메모리 | 효율 | 오버헤드 |

## 메모리 구조
```
Primitive:
Stack: int a = 10  → 10 직접 저장

Reference:
Stack: String str ──→ Heap: "Hello" 객체
```

## 선택 기준

**Primitive 사용**:
- 단순 값 저장
- 성능 중요
- 메모리 절약

**Reference 사용**:
- 복잡한 데이터
- null 필요
- 객체 기능 활용
